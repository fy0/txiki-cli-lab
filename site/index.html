Tiny runtime. Release-first attitude.

Ship a CLI playground that feels small, fast, and strangely powerful.

txiki-cli-lab is a compact txiki.js experiment that compiles into cross-platform binaries, refreshes a rolling dev release on every push, and still gives you room to run JavaScript, TypeScript, WASM, and tiny HTTP demos from one executable.

Chapter 01

CLI experiments usually die in folders.

A pile of scripts, a runtime that only works on your machine, and release notes that exist only in your head. txiki-cli-lab turns that mess into a repeatable shipping loop.

Chapter 02

Every push becomes evidence.

The project builds Linux x64, Linux arm64, and Windows x64 binaries, updates a shared dev release, and keeps a versioned path for tagged drops. Less ceremony, more proof.

Chapter 03

Play with scripts without abandoning binaries.

Run local JS, on-demand TypeScript, tiny WASM modules, and local HTTP demos from the same tool. It feels like a release candidate and a toybox at the same time.

What it runs

A tiny launcher with multiple personalities.

JavaScript first

Run local .js and .mjs modules directly, with an injected runtime context for quick experiments.

txiki-cli-lab run examples/echo-context.js

TypeScript when available

Use esbuild from PATH to transpile .ts and .mts on demand. If it is missing, the binary tells you exactly what to do.

txiki-cli-lab run examples/hello.ts demo mode

WASM with no theatrics

Load a .wasm module directly or orchestrate it from JS. Great for proving tiny native-ish ideas without a full app shell.

txiki-cli-lab run-wasm examples/answer.wasm

Local HTTP demos

Turn a script into a small web endpoint with one command. Useful for promo pages, fixtures, and tiny web toy servers.

txiki-cli-lab run-http examples/http-hello.js 3000

Release loop

One repo, two speeds.

01

Push to main

Builds all targets, uploads workflow artifacts, and refreshes a shared dev prerelease tag.

02

Tag a version

Uses the same build path, but promotes assets into a clean versioned release for keeps.

03

Ship proof, not promises

The smoke suite validates command behavior before artifacts leave the workflow.
